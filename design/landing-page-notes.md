# Minicode landing page direction

## Locked hero

**Headline**
A coding agent for local models

**Subhead**
minicode helps coding models read less, reason better, and use fewer tokens.

## Messaging thesis

The page should explain minicode as more than a code-map feature. The deeper story is context conservation:

- long context is not free
- whole-file reads bloat prompts and dilute attention
- small/local models suffer the most, but API-backed workflows also pay in latency and token cost
- minicode gives agents a structural way to navigate code through symbols, dependencies, and targeted retrieval

## Page structure

1. Hero
2. Problem: long context is not a free lunch
3. How it works: structural map, dependency cone, structural navigation, preserved reasoning budget
4. Why minicode is different
5. Use cases
6. Closing CTA

## Visual direction

- technical, restrained, open-source appropriate
- dark/clean product-tool aesthetic or crisp technical landing page
- emphasize flow: codebase -> structural map -> specialized tools -> better output
- avoid SaaS fluff or vague hype language

## Notes

This first mockup is intended as a direction-setting artifact, not final polish.
