Why it matters

Long context is not a free lunch

Most coding agents still treat a codebase like a pile of text. They read entire files, bloat the prompt, and burn tokens before the model has even reached the part of the task that requires judgment. The pain is most obvious on local models, but frontier models pay for it too in latency, cost, and diluted attention.

Default loop

Read everything and hope the right details survive

  • attention gets diluted across irrelevant lines
  • latency and token cost climb quickly
  • smaller local models lose headroom fastest
Minicode loop

Navigate structurally and keep the prompt lean

  • retrieve bodies, types, and adjacent dependencies on demand
  • increase signal density instead of total context volume
  • preserve more of the context window for reasoning
How it works

AST-indexed context optimization for code agents

minicode is built around a simple workflow: index structure first, retrieve narrowly, and spend the remaining context budget on the parts of the job that actually need thought.

01
orientation

Build a structural map

Parse the project into symbols, signatures, and relationships so the agent starts with codebase structure instead of guesswork.

02
retrieval

Follow the dependency cone

Pull the method body, types, and nearby dependencies that matter for the task instead of dragging in the whole file.

03
navigation

Walk code structurally

Use symbol- and dependency-aware tools to move through a codebase by relationships, not by linear scrolling.

04
reasoning

Preserve reasoning budget

Keep the prompt lean so more of the available context can go toward planning, editing, and validation.

What makes it different

minicode is not just a code map

The project is really about context conservation: giving models a more structured interface to code so they can spend less time reading and more time solving, while leaving room for plugins and reusable runtime integrations.

Born from local-model constraints

The first problem was making smaller models viable for coding work, but the same structural advantages carry over to frontier-model workflows too.

Use cases

Where it shines

minicode is strongest when the task is narrow, the codebase is non-trivial, and every extra token in the prompt has a cost.

Scoped engineering work

Bug fixes, method implementations, refactors, and targeted edits where a small set of symbols drives the task.

Cost-sensitive frontier workflows

Useful when API spend and response latency matter, even if the model itself has a large context window.

Local-model coding loops

Especially compelling when working on consumer hardware and every bit of prompt efficiency affects quality.

Structural code navigation

Best when the right answer lives in code relationships rather than in a single obvious file or folder.

Read less. Reason better. Use fewer tokens.

minicode is built on a simple bet: coding models perform better when you give them less context, but better context.
